How it works

Receipt
21 fields: model_id, input_hash, output_hash, proof, timestamp, provider signature. Hash the canonical bytes with SHA-256 → receipt_id. Tamper any field, the hash breaks.
Transparency log
Transfers appended to Merkle trees. Domain-separated leaf hashing (0x00 prefix), internal nodes (0x01 prefix). Every entry has an inclusion proof. Query any log to verify.
Witnessing
3 independent log servers (Nuremberg, Helsinki × 2). Each countersigns the others' checkpoints. 2-of-3 threshold. Rollback detection. No consensus needed.

Why not blockchain?

Same model as Certificate Transparency (RFC 6962). Append-only logs + cross-witnessing = auditability without consensus. Single log does O(1) append in milliseconds. Blockchain gives you liveness guarantees you don't need for receipts — the proof is self-contained, the log just prevents double-spends.

Provenance DAGs

Receipts reference parent receipts by hash. Agent B uses Agent A's output → B's receipt includes A's receipt_id. Change A and B's hash breaks. The full supply chain is cryptographically linked and independently verifiable.

Try it

Submit a receipt

pip install requests cryptography
curl -sL https://raw.githubusercontent.com/Henry-Shelton/tesseravcr/main/tessera-network/join_network.py | python3 -

Generates Ed25519 keypair, creates receipt, submits transfer, gets Merkle proof. 120 lines, no magic.

Query the API

curl https://log1.tesseravcr.org/v1/health
curl https://log1.tesseravcr.org/v1/checkpoint
curl https://log1.tesseravcr.org/v1/entry/28
curl https://log1.tesseravcr.org/v1/proof/28

Run your own log

curl -fsSL https://raw.githubusercontent.com/Henry-Shelton/tesseravcr/main/tessera-rust/tessera-log-server/deploy/setup.sh | sudo bash

What's running

Live: Receipt schema, canonical serialization, Ed25519 signing, Merkle tree construction, transparency logs, double-spend detection, witnessed checkpoints, provenance DAG persistence, HTTP API.

Tested not enforced: Trust quotient (effective stake computation), royalty cascade algorithm.

Not implemented: ZK/TEE proof verification (backend-agnostic interface defined, no backends integrated), payment rails, SDK wrappers.

Rust implementation byte-identical to Python reference. 52 tests passing. Source on GitHub.
