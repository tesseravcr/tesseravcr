The problem

AI agents can't trust each other

Your agent sends $5 to a stranger's model for "GPT-4 summarisation." Did that model run? Were the right inputs used? Could the provider charge you twice? Today there's no way to know.

Tessera VCR creates a cryptographic receipt for every transaction. It's like a bank statement that three independent servers countersign — except it's open, verifiable by anyone, and mathematically impossible to forge.

How it works

Three properties, independently verifiable

1. The receipt can't be faked

Every receipt is signed by the provider's private key. Change a single character and the signature breaks. The receipt ID is a fingerprint of all fields — if anything is altered, the fingerprint won't match.

Same cryptography as SSH keys and cryptocurrency wallets.

2. The record can't be erased

Receipts are appended to Merkle trees — a data structure where removing any entry changes the root hash. Three independent servers in different countries each maintain their own tree and countersign each other's state. Rewriting history requires compromising all three simultaneously.

Same architecture as Certificate Transparency — how your browser verifies HTTPS certificates.

3. Money can't be spent twice

Each receipt has one owner at a time. Transferring it requires the current owner's signature. Try to transfer it again and the server rejects it immediately. Try on a different server — cross-verification catches it.

No blockchain needed. Transparency logs + witnessing is sufficient.

Developers

Open API, no auth required

Every receipt, proof, and checkpoint is queryable via HTTP.

# Health check
curl https://log1.tesseravcr.org/v1/health

# Latest signed Merkle root
curl https://log1.tesseravcr.org/v1/checkpoint

# Specific receipt with provenance
curl https://log1.tesseravcr.org/v1/entry/28

# Mathematical proof of inclusion
curl https://log1.tesseravcr.org/v1/proof/28
