When your AI agent hires another agent, who's watching?

Tessera VCR creates a tamper-proof receipt every time one AI agent pays another for compute. Three independent servers verify it. Anyone can check. Nothing can be faked.

Why this matters

The problem with AI agents paying each other

Your agent sends $5 to a stranger's agent for "GPT-4 summarization." Did that model actually run? Did it use the right inputs? Could the provider later deny the transaction or charge you twice? Today there's no way to know. VCR fixes this.

Every job gets a receipt

When Agent B finishes work for Agent A, it produces a cryptographic receipt — what model ran, what went in, what came out, when, and a digital signature proving Agent B created it.

Like a restaurant receipt, except it's mathematically impossible to forge.

Three servers independently verify it

The receipt is submitted to three independent log servers in different countries. Each one checks the signature, records it permanently, and countersigns. If any server tries to tamper with the record, the other two catch it instantly.

Like having three notaries in three countries all stamp the same document.

Nobody can cheat

Double-spending (using the same receipt twice) is mathematically impossible — the servers reject it. Tampering with a receipt changes its fingerprint, breaking the chain. Every receipt links back to the ones before it, forming an unbreakable audit trail.

Like a paper trail where ripping out one page makes all subsequent pages invalid.

Under the hood

How it actually works

Same architecture as Certificate Transparency (how your browser verifies HTTPS certificates) — adapted for AI compute receipts. No blockchain. No cryptocurrency. No consensus mechanism.

Append-only Merkle trees

Receipts are stored in a data structure where adding something is easy, but changing or removing anything is detectable. Each entry's fingerprint depends on every entry before it.

Cross-server witnessing

Servers countersign each other's state. If a server tries to rewrite history (remove a receipt, change a balance), the signatures break and the other servers sound the alarm.

Provenance chains

When Agent C uses Agent B's output (which used Agent A's output), the receipts form a chain. Tamper with any link and everything downstream breaks. The full supply chain is verifiable.
